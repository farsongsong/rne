어떻게 숫자로 볼까

사람 눈에는 하나의 색이지만, 컴퓨터 안에서는 서로 다른 방식의 숫자 3세트로 동시에 저장됩니다.

RGB

빨강·초록·파랑 빛의 밝기(0~255)로 색을 표현. 가장 기본적인 방식이지만 조명이 바뀌면 세 값이 함께 흔들린다.

HSV

색상(어떤 색인지)·채도·명도를 분리해서 표현. 색상 값은 밝기 변화에 비교적 흔들리지 않는다.

Lab

사람의 눈이 색을 인식하는 방식을 본뜬 색공간. 밝기와 색의 위치를 나누어 표현한다.

🌲 RandomForest

100명에게 물어보고
다수결로 정하기

비유 — 색상 값을 100명의 판정단에게 나눠주고, 각자 일부 단서만 보고 "이건 pH 얼마 같아요"라고 추측하게 한 뒤, 100명의 답을 평균 내는 것과 같습니다. 한 사람(트리)이 틀려도 전체 평균은 안정적입니다.
  • 여러 개의 '의사결정나무'가 서로 다른 데이터로 독립적으로 학습
  • 각 나무의 예측을 평균 내어 최종 pH 결정
  • 어떤 색상 값이 예측에 가장 중요한지도 함께 계산 가능 (R값 74.4%)

가장 비슷한 색을
찾아 평균 내기

비유 — 270개의 색상 표본이 담긴 색종이 상자가 있다고 생각해보세요. 새로운 색이 들어오면, 상자에서 가장 비슷한 색종이 5장을 찾아 그 종이에 적힌 pH 값 5개를 평균 냅니다. 학습 과정 없이 비교만으로 예측합니다.
  • 색이 얼마나 비슷한지는 '거리'라는 숫자로 계산 (유클리드 거리)
  • 서버 없이 브라우저 자바스크립트만으로 실시간 실행 가능
  • 데이터가 많아질수록 정확해지지만, 계산량도 함께 늘어남
실제 사용한 코드 (index.html 기반)
function knn(q, k = 5) { // 저장된 270개 데이터와 거리 계산 const ds = DB.map(row => ({ d: distance(q, row), ph: row[9] })); // 가까운 순서로 정렬 후 5개만 선택 ds.sort((a,b) => a.d - b.d); const top = ds.slice(0, k); // 5개의 pH 값을 평균 내어 반환 return average(top.map(x => x.ph)); }

데이터와 화학 공식을
동시에 배우기

비유 — 시험공부를 할 때 정답지(데이터)만 외우는 학생과, 정답지와 함께 공식(화학 원리)까지 이해하는 학생의 차이입니다. 후자는 처음 보는 문제(새로운 조명·환경)에서도 원리를 근거로 더 안정적으로 답할 수 있습니다.
  • 산성에서 R값이 크고, 염기성에서 B값이 크다는 원리를 수식으로 정리
  • 이 수식을 '손실 함수'에 추가해 신경망이 원리에 어긋나지 않게 학습
  • 데이터가 충분한 지금은 효과가 크지 않지만, 데이터 부족·환경 변화 시 안정성 기여 기대
색상 기준식 (물리 손실의 근거)
// 산성일수록 R↑, 염기성일수록 B↑ 라는 원리를 식으로 표현 pH = 7 - 5 * (R - B) / (R + G + B)
개념 코드 · 두 가지 손실을 함께 줄인다
# 1) 데이터와 비교한 오차 data_loss = mse(model(X), y_true) # 2) 화학 공식과 비교한 오차 physics_loss = mse(model(X), formula(X)) # 두 오차를 동시에 줄이도록 학습 total_loss = data_loss + 0.01 * physics_loss

성능을 확인하는 방법

R²와 MAE, 무엇을 보는 숫자인가

MAE — 평균절대오차

MAE = 평균( |예측 pH − 실제 pH| )

가장 직관적인 지표입니다. "평균적으로 pH가 얼마나 틀렸는가"를 그대로 보여줍니다. 본 연구의 MAE 0.032는 "평균적으로 pH 0.032 정도 차이 난다"는 뜻입니다.

R² — 결정계수

R² = 1 − (모델이 설명 못한 변동 / 전체 변동)

R²는 오차 그 자체가 아니라, pH가 오르내리는 패턴을 모델이 얼마나 잘 따라가는지를 0~1로 나타낸 값입니다. 1이면 완벽히 설명, 0이면 "그냥 평균값만 찍는 것"과 다름없다는 뜻입니다. 1−R²가 작을수록 설명하지 못한 부분(=오차와 관련된 몫)이 적다는 의미라서, 오차 이야기와 함께 자주 등장합니다.

알고리즘이 스스로를 확인하는 법

5번 시험 보고
평균을 낸다 — 5-Fold 교차검증

데이터를 5묶음으로 나눈 뒤, 매번 다른 1묶음을 시험 문제로 남기고 나머지 4묶음으로 공부(학습)시킵니다. 이걸 5번 반복해 5개의 점수(R², MAE)를 낸 뒤 평균과 표준편차를 봅니다.
  • 단순 80:20 한 번 나누기보다 훨씬 엄격한 검증
  • "운 좋게 쉬운 데이터가 시험에 나온 것"인지 걸러낼 수 있음
  • 5번의 점수가 서로 비슷할수록(표준편차 작을수록) 안정적인 모델
